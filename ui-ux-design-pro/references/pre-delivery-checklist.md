# Pre-Delivery Checklist

## Visual quality
- Is hierarchy obvious in 3 seconds?
- Are spacing, radius, borders, and shadows consistent?
- Are icons aligned and stroke weights consistent?

## Interaction
- Are all key components covered for hover, focus, active, disabled, loading, empty, success, and error?
- Is destructive behavior clearly differentiated?
- Is feedback immediate and specific?

## Light / dark mode
- Do both themes preserve contrast and semantic meaning?
- Do charts, dividers, overlays, and disabled states still read correctly?

## Layout
- Does the layout hold at common breakpoints?
- Are sticky regions helpful rather than obstructive?
- Is density appropriate for the user role and task frequency?

## Accessibility
- Are contrast, focus, labels, tab order, and target sizes acceptable?
- Are error messages and validation recovery clear?

## Handoff
- Are component names, variants, token references, and usage rules documented?
- Can engineering identify spacing and behavior without guessing?
