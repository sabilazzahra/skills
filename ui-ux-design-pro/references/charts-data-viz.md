# Charts & Data Viz

25 chart types with use-case guidance.

## Catalog
1. **Bar Chart** — Compare discrete categories; best for magnitude ranking.
2. **Column Chart** — Time-bucket comparisons when vertical scanning is preferred.
3. **Line Chart** — Show change over time with continuous intervals.
4. **Area Chart** — Show cumulative trend or contribution over time; avoid too many series.
5. **Stacked Bar** — Compare composition across categories.
6. **100% Stacked Bar** — Show proportional composition only.
7. **Grouped Bar** — Compare multiple series per category without stacking ambiguity.
8. **Scatter Plot** — Reveal correlation, clusters, and outliers.
9. **Bubble Chart** — Add a third variable via size; use sparingly.
10. **Histogram** — Show frequency distribution.
11. **Box Plot** — Summarize spread, quartiles, and outliers.
12. **Heatmap** — Surface intensity across 2 dimensions.
13. **Treemap** — Show part-to-whole in constrained space.
14. **Funnel Chart** — Display stage conversion drop-off.
15. **Waterfall Chart** — Explain sequential gains and losses.
16. **Gauge** — Only for executive-at-a-glance status with clear thresholds.
17. **Bullet Chart** — Compact progress vs target with benchmark context.
18. **Donut Chart** — Use only for small part-to-whole sets with direct labels.
19. **Radar Chart** — Only for expert comparison of fixed dimensions.
20. **Cohort Retention Chart** — Track retention behavior over time cohorts.
21. **Calendar Heatmap** — Show daily activity intensity and streaks.
22. **Sankey Diagram** — Visualize flow between stages or systems.
23. **Network Graph** — Map relationships when topology matters.
24. **Map Choropleth** — Regional comparison using normalized values.
25. **Table + Inline Sparkline** — Best for operator workflows that need detail and trend together.

## Best practices
- Label axes and units before styling refinements.
- Avoid pie/donut when precise comparison matters.
- Use tooltips as secondary detail, not the only way to read values.
- Keep color meaning stable between charts and status semantics.
- Pair charts with summaries and filters when decisions depend on them.
