# UI/UX Design Pro

Use this skill to produce design directions that are crisp, implementation-aware, and suited for modern product UI.

## Workflow

1. Identify the product type, user goal, platform, and target stack.
2. Pick a visual direction from `references/styles.md`.
3. Choose a palette strategy from `references/color-system.md`.
4. Match typography from `references/typography.md`.
5. Apply product-specific patterns from `references/product-types.md`.
6. Validate flows and states against `references/ux-guidelines.md`.
7. For analytics-heavy UI, use `references/charts-data-viz.md`.
8. Tailor output to implementation constraints with `references/stacks.md`.
9. Finish with `references/pre-delivery-checklist.md` and `references/rule-categories.md`.

## Output standard

Default to **design UI** output.

Include, when relevant:

- Objective
- Visual direction
- Layout structure
- Component/system decisions
- Interaction states
- Accessibility notes
- Developer handoff notes
- QA checklist

## Selection rules

- Prefer clarity over novelty for B2B SaaS and internal tools.
- Use stronger visual styles only when they support brand positioning.
- Keep product type, data density, and engineering constraints visible in every recommendation.
- Avoid generic inspiration dumps; convert references into concrete UI decisions.

## Reference routing

- Visual style exploration → `references/styles.md`
- Palette selection and semantic color behavior → `references/color-system.md`
- Font pairing and scale → `references/typography.md`
- Product-specific UI reasoning → `references/product-types.md`
- UX rules, states, forms, navigation, and motion → `references/ux-guidelines.md`
- Data-heavy interfaces → `references/charts-data-viz.md`
- Stack-aware implementation notes → `references/stacks.md`
- Final QA before delivery → `references/pre-delivery-checklist.md`
- Decision priority, anti-patterns, and rule severity → `references/rule-categories.md`
